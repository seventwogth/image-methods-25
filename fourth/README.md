# ЛР 3.5: обратная трассировка лучей с Intel Embree 3

## Цель работы

Сформировать изображение трехмерной сцены методом обратной трассировки лучей с учетом локального освещения, зеркальной составляющей и теней. Для ускорения пересечения лучей со сценой используется Intel Embree 3.

## Что реализовано

- C++17-проект с отдельными модулями для математики, сцены, рендерера и изображения;
- трассировка первичных лучей камеры;
- пересечение со сценой через `rtcIntersect1`;
- тени через shadow rays и `rtcOccluded1`;
- RGB-освещение по модели Ламберта + Блинн-Фонга;
- ограничение RGB в диапазон `0..1`;
- gamma correction перед записью PPM;
- экспорт результатов в CSV и PPM;
- русский консольный вывод в стиле предыдущих лабораторных.

## Сцена

Сцена состоит из треугольных мешей:

- пол из двух треугольников;
- задняя стена из двух треугольников;
- боковая стена из двух треугольников;
- пирамида из шести треугольников.

Используются два цветных точечных источника света:

- теплый источник сверху-слева;
- холодный источник справа.

Материалы содержат:

- диффузный цвет;
- зеркальный цвет;
- коэффициенты `kd`, `ks`;
- показатель блеска `shininess`.

## Выходные файлы

После запуска в директории `fourth/output` создаются:

- `render.ppm`
- `input_camera.csv`
- `input_lights.csv`
- `input_triangles.csv`
- `render_summary.csv`
- `control_rays.csv`

Все CSV сохраняются с разделителем `;`.

## Требование к Embree

Проект ожидает установленный Intel Embree 3. В `CMakeLists.txt` сначала используется `find_package(embree 3 CONFIG QUIET)`, а затем выполняется ручной поиск заголовков и библиотеки через `EMBREE_DIR`/`EMBREE_ROOT`.

Если Embree не найден, конфигурация CMake завершается понятной ошибкой.

## Алгоритм

1. Создать учебную сцену из треугольников.
2. Построить `RTCScene` и загрузить в него геометрию мешей.
3. Для каждого пикселя построить луч камеры.
4. Найти ближайшее пересечение через `rtcIntersect1`.
5. Если пересечение найдено, вычислить нормаль и локальное освещение.
6. Для каждого источника построить shadow ray.
7. Проверить затенение через `rtcOccluded1`.
8. Просуммировать диффузную и зеркальную составляющие.
9. Ограничить цвет в диапазон `0..1`.
10. Применить gamma correction и сохранить `PPM`.
11. Выгрузить входные данные и сводку в CSV.

## Примечание

В этой реализации не используется глобальное освещение и не выполняется многократный путь луча. Это соответствует первому этапу работы: прямое локальное освещение и тени на основе Embree.
